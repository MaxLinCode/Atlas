# Product Docs

This directory is the source of truth for product decisions.

## Suggested structure

- `mvp-requirements.md`: locked scope for the current MVP
- `vision.md`: longer-term product direction beyond the MVP
- `roadmap.md`: sequenced post-MVP feature expansion

## Boundaries

- Put user-facing behavior, scope, and acceptance criteria here.
- Keep technical architecture in `docs/architecture.md`.
- Record irreversible architecture or infrastructure choices in `docs/decisions/`.
- Keep temporary execution status in `docs/current-work.md`.
