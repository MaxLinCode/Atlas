# Product Vision

Atlas starts as a Telegram-first brain-dump scheduler and can grow into a more capable execution assistant over time.

## Long-term direction

- Accept messy input with near-zero friction.
- Turn raw thoughts into increasingly usable tasks.
- Help the user move from capture to execution with less overhead.
- Expand from simple scheduling toward more adaptive planning and guidance.

## Deferred capabilities currently in view

- Adaptive task breakdown based on user capacity
- Smarter scheduling that uses fit, timing, and workload heuristics
- Automatic rescheduling when work slips
- Guidance about what to do next
- Future calendar integration and broader delivery surfaces

This file should stay high-level. Concrete scope decisions belong in `mvp-requirements.md` and later roadmap docs.
